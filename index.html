Reconnaissance

In dieser Phase führen wir eine erste Aufklärung durch, um Informationen über das Zielsystem zu sammeln. Dies beinhaltet das Scannen des Netzwerks, um aktive Hosts zu identifizieren, und das Überprüfen offener Ports und Dienste.
Analyse: Die Reconnaissance ist entscheidend, um ein Verständnis der Netzwerkstruktur und der potenziellen Angriffsfläche zu gewinnen.
Empfehlung: Verwenden Sie verschiedene Tools und Techniken, um ein umfassendes Bild des Zielsystems zu erhalten.

┌──(root㉿cyber)-[~]
└─# arp-scan -l
192.168.2.112 08:00:27:69:3b:76 PCS Systemtechnik GmbH

arp-scan -l listet alle Geräte im lokalen Netzwerk auf. Hier sehen wir, dass sich ein Gerät mit der IP-Adresse 192.168.2.112 im Netzwerk befindet.
Analyse: ARP-Scan hilft, aktive Hosts im lokalen Netzwerk zu identifizieren. Die MAC-Adresse kann zur Geräteerkennung verwendet werden.
Empfehlung: Verwenden Sie ARP-Scan, um schnell einen Überblick über die aktiven Geräte im Netzwerk zu erhalten.

Web Enumeration

Wir verwenden Gobuster, um versteckte Verzeichnisse und Dateien auf dem Webserver zu finden.
Analyse: Web Enumeration ist wichtig, um versteckte Inhalte und potenzielle Schwachstellen aufzudecken, die nicht direkt zugänglich sind.
Empfehlung: Verwenden Sie verschiedene Wordlists und Erweiterungen, um eine umfassende Suche durchzuführen.

┌──(root㉿cyber)-[/home/cyber/Downloads]
└─# gobuster dir -u http://192.168.2.112 -w /usr/share/seclists/Discovery/Web-Content/directory-list-2.3-medium.txt -e -x .git,php,html,xml,zip,7z,tar,bak,sql,py,pl,txt,jpg,jpeg,png,js,aac,ogg,flac,alac,wav,aiff,war,jse,jar,asp,aspx,csv,rtf,doc,docx,dsd,mp3,mp4,mkv,log,sh,dll,exe -t 100
http://192.168.2.112/index.php (Status: 200) [Size: 1621]
http://192.168.2.112/assets (Status: 403) [Size: 153]

Gobuster fand die Datei index.php und das Verzeichnis assets. Das Verzeichnis assets ist nicht zugänglich (Status 403).
Analyse: Gobuster hilft, versteckte Dateien und Verzeichnisse auf dem Webserver zu finden.
Empfehlung: Überprüfen Sie die Konfiguration des Webservers und stellen Sie sicher, dass keine unnötigen Dateien oder Verzeichnisse öffentlich zugänglich sind.

http://192.168.2.112/index.php

Im Quellcode von index.php wurde ein Kommentar gefunden: "Kung Fu Panda".
Analyse: Kommentare im Quellcode können Hinweise auf die Funktionalität oder den Zweck der Anwendung geben.
Empfehlung: Überprüfen Sie alle Kommentare im Quellcode auf sensible Informationen.

┌──(root㉿cyber)-[~]
└─# echo "S3VuZ19GdV9QNG5kYQ" | base64 -d
Kung_Fu_P4nda

Der Base64-String "S3VuZ19GdV9QNG5kYQ" decodiert zu "Kung_Fu_P4nda".
Analyse: Base64-Strings werden oft verwendet, um Daten zu verschlüsseln oder zu kodieren.
Empfehlung: Decodieren Sie alle Base64-Strings im Quellcode, um potenzielle Informationen aufzudecken.

Privilege Escalation

Wir überprüfen, welche sudo-Rechte der Benutzer "po" hat.
Analyse: Sudo ermöglicht es Benutzern, Befehle mit den Rechten eines anderen Benutzers auszuführen.
Empfehlung: Beschränken Sie die sudo-Rechte auf das absolute Minimum, das für die Aufgaben des Benutzers erforderlich ist.

po@hostname:~$ sudo -l
We trust you have received the usual lecture from the local System
Administrator. It usually boils down to these three things:

#1) Respect the privacy of others.
#2) Think before you type.
#3) With great power comes great responsibility.

[sudo] password for po:
Sorry, user po may not run sudo on hostname.

Der Benutzer "po" darf keine sudo-Befehle ausführen.

po@hostname:~$ cat /etc/passwd | grep bash
root:x:0:0:root:/root:/bin/bash
po:x:1000:1000::/home/po:/bin/bash
oogway:x:1001:1001::/home/oogway:/bin/bash

Wir überprüfen die vorhandenen Benutzer mit einer Bash-Shell. Wir finden den Benutzer "oogway".

po@hostname:~$ cat /etc/sudoers.d/po
po HackMyVM = (oogway) NPASSWD: /bin/bash

Der Benutzer "po" darf den Befehl /bin/bash als Benutzer "oogway" auf dem Host "HackMyVM" ohne Passwort ausführen.

Wir versuchen, den Befehl auszuführen.

po@hostname:~$ sudo -u oogway -h HackMyVM /bin/bash
sudo: unable to resolve host HackMyVM: Name or service not known
